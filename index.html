Domain & Hosting

For this project, I chose to host my website using GitHub Pages, a free and easy-to-use hosting service that allows me to deploy static websites directly from a repository. I used a GitHub subdomain (e.g., myusername.github.io) since it provided a simple and effective solution without requiring a paid domain.

Setting up the domain involved configuring my repository, enabling GitHub Pages, and ensuring that my index.html file was properly placed in the root directory or the designated branch.

Website Setup

I built this website using HTML for the structure, CSS for styling, and kept it simple without additional frameworks. Here’s how I approached the development:

  • HTML: I structured the webpage using elements like headings, paragraphs, and lists to organize the content logically.
  • CSS: I applied basic styling, including background colors, font choices, and a responsive container for a clean layout.
  • JavaScript (optional): Though not included here, I considered adding JavaScript for interactive features in the future.

Networking Aspects

Making my site accessible online required an understanding of key networking concepts. Here’s how they played a role:

Domain Name System (DNS)

Since I used GitHub Pages, my website’s domain was automatically mapped to a GitHub-provided URL. If I wanted a custom domain, I could configure a CNAME record in the DNS settings to point to my GitHub-hosted site.

Common DNS records that could be relevant include:

  • A Record: Used if I were pointing a custom domain to an IP address.
  • CNAME Record: Directs a subdomain (e.g., www.example.com) to my GitHub Pages site.

IP Addresses

Each website is accessed via an IP address, even if users type a domain name. My site, hosted on GitHub Pages, resolves to GitHub’s IP addresses, ensuring that visitors can reach my content.

Networking Protocols

Several protocols were involved in making my site available:

  • HTTP/HTTPS: My site is accessible over HTTPS, ensuring secure data transfer.
  • TCP/IP: This protocol suite enables communication between my browser and GitHub’s servers.
  • DNS Protocol: Translates my GitHub Pages domain into an IP address.

By setting up this website, I gained hands-on experience with hosting, networking concepts, and web development fundamentals.
